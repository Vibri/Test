Pure functions
---------------
- characteristics
	- 1. The output of a pure function depends only on (a) its input parameters and (b) its internal algorithm.
	- 2. A pure function has no Side Effects
	
public class PureFunction {
	public int sum(int a, int b) {
		return a + b;
	}
}
	
public class ImpureFunction {
	private int value = 0;
	
	public int add(int nextValue) {
		this.value += nextValue;
		return this.value;
	}
}

- how to detect pure function - When you substitude the calls of function the result must be same (also the logging)
- for example - the priinting on the console is not pure function. It is the side effect.
- No Side Effect

Tail Call Optimization
----------------------
(Tail Call Recursion)
	
	Normal recursion
	- public static long reFactorial(int n) {
		if(n <= 1) {
			return 1;
		}
		
		return n*reFactorial(n-1);
	}
	
	Tail Recursion
	- public static long tailRecursiveFactorial(int n, int a) {
		if ( n <= 1) {
			return a;
		}
		else 
		return tailRecursiveFactorial(n-1, n*a);
	 }
	 
	 - Compiler can convert tail recursion into a Pure Iteration
	 Tail Recursion => Pure Iteration

------------------------
The strategy design pattern is used when we have multiple solution or algorithms
to solve a specific task and client decides the specific implementation

DECORATOR DESIGN PATTERN
------------------------
The Decorator design pattern is used to modify functionality of object at runtime without affecting other instances of the same class.

- in functional programming we don't modify the instance state, but we return a new instance
